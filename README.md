# Open2FA 前端项目

Open2FA 是一个基于 Vue 3、TypeScript 与 Arco Design 的双因素认证（2FA）配置台前端项目，为离线设备提供动态口令二维码的统一管理能力。当前实现了 v1.0.0 的核心功能，包括用户认证、角色权限控制以及设备二维码的生成与展示。

## 功能特性

- 🔐 **用户认证与角色管理**：内置超级管理员与普通用户两种角色，支持基础登录校验。
- 🧩 **离线设备管理**：以设备唯一序列号为核心，自动生成兼容谷歌与微软验证器的 TOTP 二维码。
- 👀 **权限区分**：普通用户可查询二维码与密钥详情，超级管理员额外具备添加设备的权限。
- 💾 **本地持久化**：会话信息与设备列表保存在浏览器存储中，方便演示与调试。

## 快速开始

```bash
npm install
npm run dev
```

启动后访问 `http://localhost:5173` 即可体验。

> 💡 本项目不包含真实后端，仅用于演示 2FA 配置平台的前端界面与交互流程。

## 目录结构

```
├── public/               # 静态资源
├── src/
│   ├── assets/           # 全局样式与静态资源
│   ├── components/       # 复用组件（设备表格、添加设备弹窗）
│   ├── router/           # 路由配置与守卫
│   ├── store/            # Pinia 状态管理（认证、设备）
│   ├── utils/            # 工具方法（TOTP URI 生成等）
│   ├── views/            # 页面级组件（登录、设备管理）
│   └── main.ts           # 应用入口
├── index.html
├── package.json
└── tsconfig*.json
```

## 设备二维码说明

- 二维码遵循 `otpauth://` URI 规范，可直接被谷歌验证器、微软身份验证器等常见 2FA 工具识别。
- 新增设备时会生成 20 位 Base32 密钥，同时提供密钥文本与二维码两种形态便于备份。
